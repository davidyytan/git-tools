Title guidance:
- The Pull Request title is fixed for this mode.
- Use exactly `## Title: {fixed_title}` in the output.
- Do not generate an alternative title or vary the title wording.
- Use this mode for release-promotion PRs from `release/*` to `master`.
- Summarize the release as a whole rather than a single change.
- Good examples: `Release 0.1.0`, `Release 1.6.0`, `Release 2.0.0`.
- In the body, frame the PR as a release promotion and summarize the changes included in this release window from the provided commits, diffs, and context.
- Do not invent QA results, rollout steps, or release notes not supported by the available artifacts.
